# Release Gate v2

- Version: `v2.0.0-rc.2`
- Date (UTC): `2026-03-09`
- Owner: `Codex`
- Network: `polkadot-hub-testnet`

## 1. Scope

Release Gate v2 closes `T21~T27` with three goals:
- deterministic frontend CI/typecheck execution,
- auditable deployment evidence and post-deploy operations,
- production-oriented reliability/security/release documentation.

## 2. Change Log

1. Engineering reliability
- Added `frontend/tsconfig.typecheck.json` to decouple `pnpm typecheck` from `.next/types` generation timing.
- Frontend CI now runs single-source command `pnpm ci-check`.

2. Deployment auditability
- Archived manual on-chain deployment artifacts under `docs/deployment-artifacts/*`.
- Updated `docs/deployment-artifacts/release-gate-20260309.json` with full tx hashes, blocks, timestamps, constructor params, and verification reads.
- Added `docs/DEPLOYMENT.md` with third-party verification commands.

3. Operations automation
- Added `script/post_deploy_check.sh` for strictXcm/notifier/health/max params/allowlist sanity checks.
- Added `script/retry_failed_notify.sh` for batched FAILED scan + retry with rate limit and summary.
- Added `ops/metrics/export_weekly_metrics.sh` for weekly CSV/Markdown exports.

4. Security and release governance
- Added `docs/SECURITY_REVIEW.md` checklist and findings closure status.
- Updated `docs/METRICS.md` with executable export workflow.

## 3. Rollback Plan

1. Emergency pause path
- If notifier path degrades, governance sets `strictXcm=false` to avoid blocking withdrawals.
- If downstream instability persists, set `notifier` to a safe adapter via `setNotifier`.

2. Contract rollback path
- Keep previous deployment addresses in deployment evidence history.
- Repoint frontend `NEXT_PUBLIC_POLKASTREAM_ADDRESS` to last known good contract and redeploy frontend.

3. Data/ops rollback path
- Retry jobs are idempotent on successful records (`retryNotify` returns SUCCESS when already successful).
- Metrics exports are append-only files and can be regenerated by block range.

## 4. Release Exit Criteria

- `forge test -q` passes.
- `forge coverage --report summary` keeps `PolkaStream.sol` branch coverage >= 80%.
- `pnpm ci-check` passes and local repeatability check succeeds.
- `script/post_deploy_check.sh` returns exit code 0.
- `TASKS.md` for `T21~T27` marked done with concise summaries.
